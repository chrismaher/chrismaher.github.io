Using Z in Vim

Aug 29, 2020
3 min read

The Z script is a popular utility for directory-hopping on Unix-y systems (popular alternatives include fasd and autojump). It tracks visited directories, and, when invoked with a directory substring, jumps to a matching directory; when presented with multiple matches, it jumps to the first match from its ‘frecency’ algorithm. So if you’re in your ~/Documents directory and want to jump to your ~/go/src/ directory, you can jump by simply entering z src. Not long after adopting Z, I found myself wanting to use Z as a command in Vim to set the working directory for different tabs (a new approach I’ve taken to juggling multiple projects from a single Vim session). I first looked at what plugin functionality existed for Z-Vim integration; the primary plugin I found is appropriately named z.vim, and you may find it works for you. Since my desired feature set was both small & specific, I decided to write a command myself to add to my .vimrc.

Implementation

First, the entire .vimrc snippet:

function! s:z(arg, lin, pos)
    let list = systemlist('_z -l ' . a:arg)
    return map(list, {_, v -> substitute(v, '\S\+\s\+', '', '')})
endfunction
command! -nargs=1 -complete=customlist,<SID>z Z execute 'tcd ' . <SID>z(<q-args>, '', '')[0]

The function s:z is script-local and needs to accept three arguments (given the names arg, lin, pos in this case) to enable command-completion from a custom list, which allows us to simulate Z’s tab-completion from the Vim command line.

The next step is to pass the first argument to the shell command _z -l; z itself is actually an alias for _z 2>&1 (as seen here), and so instead of invoking the alias, we call the _z function directly and pass the -l option, which prints matches and their frecency scores to stdout instead of changing directory. Note that an -e option also exists, which should echo the best match (instead of cd’ing to it), but I’ve found in my Z shell that this doesn’t work as expected.

We then parse the results from the system call to _z -l to obtain an ordered list of the matching filenames (without their frecency scores, since we can rely instead on just the ordering) and return the list to the caller.

The final step is to create the necessary command. We enable tab-completion from a custom list using -complete=customlist,<SID>z, which will populate the completion list using our z function itself. The main body of the command will call tcd, so that we can maintain tab-specific working directories, with the result from calling our z function with the command-line argument.

Usage

The usage of this new command is pretty straightforward: just call :Z src, for instance, to set the working directory of the current tab to whatever matches src, just as if we had called z src from the shell.

And that’s it! Hopefully you find that this command saves you some keystrokes.
