Minimum Viable Optparse-Applicative

Nov 1, 2020
2 min read

There are plenty of good blog posts & examples for optparse-applicative on the web, but this example offers a minimum viable optparse-applicative program:

import Options.Applicative
import Data.String (IsString)

parser :: IsString s => Parser s
parser = strArgument mempty

parserInfo :: IsString a => ParserInfo a
parserInfo = info parser mempty

main :: IO ()
main = execParser parserInfo >>= putStrLn

The program doesn’t do much; it accepts and prints exactly one argument. It’s useful, though, in showing the most fundamental optparse-applicative building blocks:

  • As stated in the Hackage documentation, the simplest way to run an optparse-applicative program is with execParser, which has type signature ParserInfo a -> IO a. The info function, which has signature Parser a -> InfoMod a -> ParserInfo a, is the preferred way to create a ParserInfo.
  • The InfoMod type can be used to construct the help text for a parser, but for now, we just pass mempty to keep the program simple.
  • The last value we need, then, is a Parser. We can create one using one of the parser builders, which we can use to create flags, options, arguments, and commands. In this example, we create a Parser with the strArgument function, which has signature IsString s => Mod ArgumentFields s -> Parser s. The Mod ArgumentFields s argument is what’s known as an option modifier; for now, we again pass mempty for simplicity.

I used top-level functions in the example in order to include type signatures, but we could also write the example in a single function using let or where bindings:

import Options.Applicative

main :: IO ()
main =
    let parser = strArgument mempty
        parserInfo = info parser mempty
    in execParser parserInfo >>= putStrLn
